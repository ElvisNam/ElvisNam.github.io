玩转RxJS（一）：RxJS简介

发表于 2019-01-13 分类于 前端笔记 ， 玩转RxJS

近一年来，前端内最推荐学习的两门技术就是 RxJS 和 TypeScript 了。

RxJS 是 Reactive Extension 这种模式的 JavaScript 语言实现，通过它你可以接触到函数响应式编程思想。但同样的，RxJS 对于入门者来说并不友好，你也可能早就听说过 RxJS 悬崖式的学习曲线。

因为业务需要，我最开始学习 RxJS 时，在网上搜索了一系列的学习文档，包括官方教程。但是在这些教程中，一上来就直接给你塞入 Subject、Subscription、Subscribe、Observable、Observer 等几个概念，对于不熟悉观察者设计模式的人来说，看的云里雾里，同时对这几个长得差不多又反反复复出现的单词搞得一脸懵。随后，教程文档直接转变为操作符的介绍，基本内容没搞懂，操作符的介绍又类似于 API 文档一样简洁，导致学习直接断层，因此很多人在此就放弃了。我也是这种情况，但是因为当时需要操作的逻辑较为简单，配合一些源码实例，搞清楚了 map、of、catchError 等几个简单操作符就够了。但随着业务逻辑复杂度越来越高，经常需要对大量异步操作进行组合处理，而之前学的不扎实，导致写出来的代码回调一层套一层，又长又臭。因此我下定决心，将 RxJS 重新再学一遍，这一次一定要弄清楚它的核心思想。

在这次的学习中，由于有了一定的使用经验，又找到了一本极佳的学习教程：程墨的《深入浅出RxJS》，学起来就感觉顺利多了。同时我也对函数式编程、响应式编程两种思想有了新的认识。因此，我打算开启这套 RxJS 的学习教程，一方面记录下自己学习过程中的心得体会，另一方面也希望通过这些文章帮助还在 RxJS 学习道路中感到困惑的朋友。本系列也是自己在阅读《深入浅出 RxJS》一书的读书笔记。
